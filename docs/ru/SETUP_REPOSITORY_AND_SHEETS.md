# Подключение репозитория и настройка Google Sheets

> Статус: рабочая версия
> Язык: русский
> Тип документа: практическая инструкция

---

## Назначение

Этот документ описывает, как подключить репозиторий Family Health OS к ChatGPT-проекту, создать Google Sheets и использовать готовые промты для оформления рабочих таблиц.

Репозиторий проекта:

```text
https://github.com/MaximKulmars/Family-Health-Os
```

Репозиторий используется как источник архитектуры, инструкций, правил, шаблонов и промтов. Рабочие данные хранятся в Google Sheets.

---

## 1. Подключение репозитория

### Вариант А. Через GitHub-коннектор

Если в ChatGPT доступен GitHub-коннектор, подключите репозиторий:

```text
MaximKulmars/Family-Health-Os
```

Проверьте, что ассистент может учитывать файлы:

```text
README.md
docs/ARCHITECTURE.md
docs/CORE_PRINCIPLES.md
docs/ru/DEPLOYMENT.md
docs/USAGE.md
docs/MASTER_PROMPT.md
```

### Вариант Б. Вручную

Если GitHub-коннектор недоступен:

1. Откройте репозиторий в браузере
2. Скопируйте содержимое `docs/MASTER_PROMPT.md`
3. Вставьте его в инструкции ChatGPT-проекта
4. Добавьте ссылку на репозиторий
5. При необходимости прикрепите ключевые `.md`-файлы к проекту

---

## 2. Создание таблиц

Создайте четыре Google Sheets:

```text
Maxim_health
Anya_health
Bella_health
Family_health
```

Правила:

- Одна личная таблица относится к одному участнику
- Семейная таблица используется как общий координационный слой
- Названия таблиц и вкладок должны совпадать буквально
- Структура должна соответствовать `docs/ru/DEPLOYMENT.md`

## 5. Базовое оформление

Для всех вкладок используется единый формат:

- Первая строка — заголовки
- Первая строка закреплена
- Включён фильтр
- Заголовки выделены жирным
- Текст переносится
- Даты оформлены как `yyyy-mm-dd`
- Время оформлено как `hh:mm`
- Для `confidence` используются значения `high`, `medium`, `low`
- Для `source` используются значения `user`, `photo`, `document`, `lab`, `app`, `doctor`, `assistant`, `manual`

Рекомендуемые служебные поля для регулярных записей:

```text
date
time
member_id
source
confidence
comment
```

## 6. Промт: подключить репозиторий

```text
Подключи репозиторий Family Health OS как главный источник правил проекта:

https://github.com/MaximKulmars/Family-Health-Os

Учитывай файлы:

README.md
docs/ARCHITECTURE.md
docs/CORE_PRINCIPLES.md
docs/ru/DEPLOYMENT.md
docs/USAGE.md
docs/MASTER_PROMPT.md

Работай по правилам репозитория. Если старые инструкции в чате противоречат репозиторию, приоритет имеет репозиторий.

После подключения кратко подтверди:

1. Какие файлы ты учитываешь
2. Какие таблицы должны быть созданы
3. Какие member_id используются
4. Какое правило хранения данных действует
```

---

## 7. Промт: создать и оформить все таблицы

```text
Нужно подготовить Google Sheets для проекта Family Health OS.

Используй список таблиц, member_id и вкладок из README.md, docs/MASTER_PROMPT.md и docs/ru/DEPLOYMENT.md.

Создай или помоги создать:

1. Личные таблицы участников
2. Семейную таблицу
3. Все обязательные вкладки
4. Заголовки в первой строке
5. Базовое оформление вкладок

Оформи вкладки так:

- первая строка — заголовки;
- первая строка закреплена;
- включён фильтр;
- заголовки выделены жирным;
- текст переносится;
- даты в формате yyyy-mm-dd;
- время в формате hh:mm;
- для confidence используются high, medium, low;
- для source используются user, photo, document, lab, app, doctor, assistant, manual.

После настройки дай короткий отчёт:

1. Какие таблицы готовы
2. Какие вкладки созданы
3. Что нужно проверить вручную
4. Есть ли риск смешивания личного и семейного слоя
```

## 8. Промт: оформить одну личную таблицу

```text
Оформи одну личную таблицу Family Health OS.

Таблица: <название таблицы>
member_id: <member_id>
profile_type: <adult | child>
child_profile: <true | false>

Создай вкладки личной таблицы по docs/ru/DEPLOYMENT.md.

Добавь базовые служебные поля там, где они нужны:

- date
- time
- member_id
- source
- confidence
- comment

Оформи вкладки:

- первая строка — заголовки;
- первая строка закреплена;
- включён фильтр;
- заголовки выделены жирным;
- текст переносится;
- даты в формате yyyy-mm-dd;
- время в формате hh:mm.

Во вкладке Members добавь только техническую стартовую строку без рабочих данных:

member_id: <member_id>
profile_type: <adult | child>
child_profile: <true | false>
table_name: <название таблицы>

После настройки дай короткий отчёт.
```

## 9. Промт: оформить семейную таблицу

```text
Оформи семейную таблицу Family Health OS.

Таблица: Family_health
member_id: family

Создай вкладки семейной таблицы по docs/ru/DEPLOYMENT.md.

Оформи вкладки:

- первая строка — заголовки;
- первая строка закреплена;
- включён фильтр;
- заголовки выделены жирным;
- текст переносится;
- даты в формате yyyy-mm-dd;
- время в формате hh:mm.

Семейная таблица используется только для общего координационного слоя: задач, ближайших проверок, решений, визитов, ресурсов и ссылок.

После настройки дай короткий отчёт.
```

## 10. Промт: проверить структуру проекта

```text
Проверь структуру Family Health OS.

Сравни фактическую структуру Google Sheets с docs/ru/DEPLOYMENT.md.

Проверь:

1. Все ли обязательные таблицы существуют
2. Все ли обязательные вкладки созданы
3. Совпадают ли названия вкладок буквально
4. Есть ли первая строка с заголовками
5. Закреплена ли первая строка
6. Включены ли фильтры
7. Используются ли правильные member_id
8. Не смешиваются ли личный и семейный слой
9. Отмечен ли детский профиль там, где это нужно

Дай отчёт в формате:

- Готово
- Нужно исправить
- Риски
- Следующий шаг
```
